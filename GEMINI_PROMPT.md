# Gemini CLI 実装指示書 — 送料計算 Web アプリ（transfer_col_system）

## あなたの役割

このリポジトリに、Next.js（App Router）製の送料計算 Web アプリを実装してください。
以下の指示に従い、**指定されたファイルを過不足なく生成**することがゴールです。

---

## 前提・制約

- **言語**: TypeScript（strict モード不要、型は最低限でよい）
- **フレームワーク**: Next.js 14 以降（App Router）
- **DB**: なし。料金テーブルは TypeScript ファイルにハードコード
- **外部ライブラリ**: 追加インストール不要。Next.js 標準機能のみ使用
- **スコープ**: ローカルで `npm run dev` して動作確認できる状態にする
- **Vercel デプロイ・テストコード・型の厳密化**: 今回のスコープ外

## Step 3 — `app/api/shipping/route.ts`

### 処理の流れ

```
1. リクエストボディから to_prefecture と box_size を取得
2. to_prefecture === "静岡県" なら zone = "同一都道府県"
   それ以外は prefectureZone[to_prefecture] で地帯を引く
3. shippingRates["中部"][zone][box_size] で送料を引く
4. 値が null または undefined なら 400 を返す
5. 正常なら { price, to_zone, is_same_pref } を返す
```

### 実装コード

```typescript
import { NextRequest, NextResponse } from "next/server"
import { prefectureZone } from "@/data/prefecture_zone"
import { shippingRates } from "@/data/shipping_rates"

export async function POST(req: NextRequest) {
  const { to_prefecture, box_size } = await req.json()

  if (!to_prefecture || !box_size) {
    return NextResponse.json({ error: "パラメータ不足" }, { status: 400 })
  }

  const isSamePref = to_prefecture === "静岡県"
  const zone = isSamePref ? "同一都道府県" : prefectureZone[to_prefecture]

  if (!zone) {
    return NextResponse.json({ error: "未対応の都道府県" }, { status: 400 })
  }

  const price = shippingRates["中部"]?.[zone]?.[box_size]

  if (price === null || price === undefined) {
    return NextResponse.json({ error: "該当なし" }, { status: 400 })
  }

  return NextResponse.json({
    price,
    to_zone: zone,
    is_same_pref: isSamePref
  })
}
```

## 動作確認用のリクエスト例

実装完了後、以下の curl で API が正しく動くか確認できる。

```bash
# 正常系：大阪府・80サイズ → 1230円
curl -X POST http://localhost:3000/api/shipping/calculate \
  -H "Content-Type: application/json" \
  -d '{"to_prefecture":"大阪府","box_size":"80"}'

# 正常系：静岡県・80サイズ → 1090円（同一都道府県）
curl -X POST http://localhost:3000/api/shipping/calculate \
  -H "Content-Type: application/json" \
  -d '{"to_prefecture":"静岡県","box_size":"80"}'

# エラー系：コンパクト・静岡県 → 400（null設定）
curl -X POST http://localhost:3000/api/shipping/calculate \
  -H "Content-Type: application/json" \
  -d '{"to_prefecture":"静岡県","box_size":"コンパクト"}'
```

## 注意事項

- `@/data/...` のパスエイリアスは Next.js デフォルトの `tsconfig.json` で有効になっている。変更不要
- `app/page.tsx` は `"use client"` ディレクティブを先頭に付けること（`useState` を使うため）
- `app/api/shipping/route.ts` はサーバーサイドなので `"use client"` は不要
- データファイル（`data/` 配下）はサーバー・クライアント両方から import 可能だが、今回は route.ts のみから参照する
