# 送料計算 Web アプリ（transfer_col_system）

## このアプリについて

農業向け SaaS「Work TerrAS」の請求書作成フローに組み込む、送料自動算出 API およびその動作確認 UI。

Work TerrAS では農産物の受注情報をもとに請求書を発行するが、現状は送料を手入力している。このアプリがその計算を自動化する。

### なぜ独立したアプリとして作るか

Work TerrAS 本体（開発担当：隆之介）とは開発タイミングを切り離し、Yuto が単独で先行実装する。完成後、隆之介が Work TerrAS 側から `fetch` で叩く形で統合する。責務を分離することで並行開発を実現し、2026年10月リリースに間に合わせる。

### このアプリの先にあるもの

```
[Work TerrAS 請求書作成画面]
        |
        | POST /api/shipping/calculate
        |   { to_prefecture: "大阪府", box_size: "80" }
        v
[このアプリ（Vercel）]
        |
        | { price: 1230, to_zone: "関西", is_same_pref: false }
        v
[Work TerrAS 請求書に送料を自動反映]
```

## API 仕様

### POST /api/shipping/calculate

**リクエスト**

```json
{
  "to_prefecture": "大阪府",
  "box_size": "80"
}
```

| パラメータ | 型 | 説明 |
|-----------|-----|------|
| `to_prefecture` | string | 着地都道府県（例：`"大阪府"`） |
| `box_size` | string | ヤマト運輸のサイズ区分（下記参照） |

**レスポンス（正常）**

```json
{
  "price": 1230,
  "to_zone": "関西",
  "is_same_pref": false
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `price` | number | 送料（円・税込） |
| `to_zone` | string | 着地地帯名 |
| `is_same_pref` | boolean | 同一都道府県フラグ（静岡→静岡のみ `true`） |

**レスポンス（エラー）**

```json
{ "error": "該当なし" }
```

HTTP ステータス `400`。`box_size` が無効、または未対応都道府県の場合。

## 運賃テーブルの考え方

### 発地固定（Phase 1）

発地は **浜松（静岡県 / 中部地帯）** に固定。

### 同一都道府県ロジック

着地都道府県が `"静岡県"` の場合のみ、同一都道府県内運賃を適用する。
中部地帯内の他県（愛知・岐阜・三重）は通常の中部→中部運賃を適用。

```
着地 === "静岡県"
  → zone = "同一都道府県"（特別運賃テーブル参照）
着地 !== "静岡県"
  → zone = prefectureZone[to_prefecture]（通常テーブル参照）
```

## 運賃改定時のメンテ手順

1. ヤマト運輸公式サイトで最新の一般運賃表を確認
2. `data/shipping_rates.ts` の数値を更新
3. `data/prefecture_zone.ts` は地帯区分変更がない限り触らない
4. `npm run dev` でローカル動作確認後、Vercel に push

> **注意**: スクレイピングによる自動取得は採用しない（利用規約違反リスク・サイト構造変更で即死するため）

## 注意事項

- コンパクトサイズの同一都道府県運賃は設定なし（`null`）。該当した場合は `400` を返す
- Phase 2 で発地を選択可能にする場合、`shipping_rates.ts` に他発地のテーブルを追加し、API の `from_zone` パラメータを追加する
